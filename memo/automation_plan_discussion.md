# 넷케이바 크롤링 자동화 및 후속 파이프라인 연동 계획 논의 (초안)

## 1. 제안된 자동화 프로세스

### 1단계: 조건부 스캔 및 크롤링 (금/토 오전에 실행)
*   **실행 시간**: 매주 금요일 오전 10:30 ~ 11:00 (일요일 경기를 위해서는 토요일 오전에 실행)
*   **실행 주기**: 모든 데이터가 완벽하게 수집될 때까지 **10분 간격**으로 스캔 및 크롤링 시도
*   **완료 조건 (Validation)**:
    1.  출마표 데이터에 `枠` (조/테두리 번호) 및 `馬番` (마번) 컬럼 데이터가 존재하는가?
    2.  모든 출전마에 대해 해당 데이터가 누락 없이 채워져 있는가?
    3.  하위 크롤링(상세 정보 등)이 실패 없이 모두 성공했는가? (실패 시 재시도 로직 포함)

### 2단계: DB 적재 및 API 이관
*   **실행 조건**: 1단계의 모든 크롤링 및 검증이 완벽하게 성공했을 경우에만 실행
*   **실행 순서**:
    1.  `run_mode_6()`: CSV 데이터를 MariaDB 임시 테이블에 업로드 (Upsert)
    2.  `run_mode_7()`: 임시 테이블의 데이터를 최종 API 테이블로 이관 (JOIN)
*   **핵심 요구사항**: 각 단계의 성공 여부를 확실하게 체크하여, 하나라도 오류가 발생하면 다음 단계로 넘어가지 않도록 강력한 예외 처리 필요.

### 3단계: 외부 AI 예측 및 음성 생성 시스템 트리거
*   **실행 조건**: 2단계(DB 이관)가 완벽하게 완료된 후 실행
*   **실행 내용**:
    1.  오늘 수집된 경기장(`meet`) 목록을 순회하며 API 호출.
    2.  **동기 대기**: API 호출 후 응답이 올 때까지 대기 (Timeout 설정 주의).
    3.  **응답 검증**: 응답받은 JSON 데이터의 `status` 값이 `"OK"` 인지 확인.
    4.  **순차 호출**: `"OK"` 확인 시 다음 경기장 파라미터를 사용하여 API 호출 반복.
*   **예외 처리 원칙**: 
    *   API 서버로부터 `"OK"` 응답을 받았다면, 외부 시스템 내부의 세부 기능 실패 여부와 상관없이 다음 경기장으로 진행함.
    *   외부 시스템 내부 오류(일부 기능 실패 등) 발생 시에는 사용자가 **관리자 페이지에서 직접 수동으로 재시도**를 수행하므로, 파이썬 스크립트 내에서는 별도의 복잡한 재시도 로직을 구현하지 않음.

## 2. 주요 고민 및 논의 사항 (User's Concerns)

### 이슈 1: 날짜(`raceDt`)와 수집 모드 분리 문제 (확정 및 반영 완료)
*   **상황**: 금요일에는 토요일 경기만, 토요일에는 일요일 경기만 순차적으로 확정됨.
*   **결정**: `all.py`의 2번 기능을 **'토요일 수집'**과 **'일요일 수집'**으로 명확히 분리함.
*   **반영 사항**: 
    *   2번: 토요일 경기 계획 수집 (Automatic Discovery)
    *   3번: 일요일 경기 계획 수집 (Automatic Discovery)

### 이슈 2: 동일 날짜, 다중 경기장(`meet`)의 API 순차 호출 및 동기화 문제 (확정 및 반영 완료)
*   **상황**: 한 날짜에 여러 경기장 데이터가 있을 때 순차적으로 AI 시스템을 가동해야 함.
*   **확정 내용**: 외부 API(`oneRaceDt.do`)는 작업이 끝날 때까지 응답을 기다리는 **동기(Synchronous) 방식**임이 확인됨.
*   **구현 로직**: 
    *   파이썬의 `requests` 라이브러리를 사용하여 API 요청.
    *   응답 JSON의 `status` 필드를 체크하여 `"OK"`일 경우에만 다음 루프(다음 경기장)로 진행.
    *   첫 번째 경기장의 AI 작업이 완전히 끝나야만 다음 경기장 요청이 전송되므로 시스템 충돌 방지 가능.

## 3. 실행 환경 및 스케줄링 방식 (확정)

### 스케줄링: 하이브리드 방식
*   **방식**: 윈도우 작업 스케줄러 + 파이썬 내부 루프
*   **상세**: 
    1.  **윈도우 작업 스케줄러**: 
        *   **금요일 10:30**: 토요일 경기 수집(`run_mode_2`)을 위한 마스터 스크립트 실행.
        *   **토요일 10:30**: 일요일 경기 수집(`run_mode_3`)을 위한 마스터 스크립트 실행.
    2.  **파이썬 내부 로직**: 스크립트 실행 후 데이터(마번, 조 등)가 완벽해질 때까지 10분 간격으로 `while` 루프를 돌며 스캔 및 크롤링 수행.
    3.  **종료**: 모든 작업(DB 이관 및 API 호출)이 완료되면 파이썬 프로세스를 스스로 종료하여 리소스 해제.

### 노트북 환경 설정
*   **전원 설정**: 노트북 덮개를 닫아도 시스템이 중단되지 않도록 제어판에서 **"덮개를 닫을 때 아무 것도 안 함"**으로 설정.
*   **절전 모드**: 자동 절전 모드를 해제하고, 안정적인 작업을 위해 전원 어댑터(충전기) 연결 권장.
